# Java 网络五子棋 (面向对象课程设计)

## 项目简介
本项目为《面向对象程序设计 (Java)》课程设计作品。我们使用纯 Java 开发了一个包含双人对战、人机对弈（贪心算法）以及数据持久化功能的高级五子棋游戏。为了践行面向对象的设计思想，我们彻底重构了原有的单文件代码，采用了 MVC 的架构进行了模块化拆分 M (Model) - 模型层  GameModel.javaV (View) - 视图层  BoardPanel.javaC (Controller) - 控制层  GomokuGame.java。

## 🌟 核心功能
* **多种博弈模式**：支持“双人对战”、“初级人机（随机下棋）”和“高级人机（启发式评分算法）”。
* **历史战绩存储**：游戏结束或关闭后，系统会自动将比分保存至本地 `score_record.properties` 文件，下次启动时自动加载，完美实现数据持久化。
* **高交互体验**：支持悔棋功能（人机模式下智能连退两步），右侧提供实时的系统运行日志滚动显示。
* **精美UI质感**：棋盘采用护眼原木色，棋子使用 `RadialGradientPaint` 径向渐变算法进行渲染，呈现出极佳的立体 3D 质感。

## ⚙️ 架构与技术栈
* **开发语言**：Java 8+
* **GUI 框架**：Java Swing (JFrame, JPanel, Graphics2D)
* **设计原则**：单一职责原则，高内聚低耦合
* **核心算法**：
  * **胜利判定**：以最新落子点为中心的四轴双向扫描算法（时间复杂度极低 O(1)）。
  * **AI 引擎**：局部最优贪心算法 + 攻防双重视角的自定义棋型权重评估。

## 🧑‍🤝‍🧑 团队分工
本项目由 3 人团队合作完成，具体职责划分如下：

* **[成员 A] 算法与核心逻辑**：负责 `GameModel.java` 与 `AIPlayer.java`。编写了基于中心扩散的胜负判定算法与 AI 的攻防打分系统，让电脑具备了拦截与绝杀的能力。
* **[成员 B] 图形界面与视图**：负责 `GomokuGame.java` 与 `BoardPanel.java`。完成了界面的垂直堆叠布局，以及物理像素到逻辑二维数组的精准坐标映射与抗锯齿渲染。
* **[成员 C / 组长] 数据持久化与统筹**：负责 `RecordManager.java`。利用 Java 的 `Properties` 类与 `try-with-resources` 语法实现了本地文件的安全读写。并主导了代码的 MVC 架构重构与 Git 版本控制管理。

## 🚀 如何运行
1. 确保您的电脑已安装 Java 运行环境 (JDK 8 或以上)。
2. 获取本仓库代码，在根目录下可以找到已经打包好的 `GomokuGame.jar` 文件。
3. **直接双击** 该 `.jar` 文件即可畅玩；
   *(或者在终端/命令行中输入以下命令运行：)*
   ```bash
   java -jar GomokuGame.jar
   ```
