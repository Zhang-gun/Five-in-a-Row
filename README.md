# Java 五子棋对弈系统 (面向对象课程设计)

## 项目简介
本项目为《面向对象程序设计 (Java)》的期末课程设计作品。我们使用原生 Java Swing 开发了一个包含双人对战、人机对弈（简单/中等/困难）以及数据持久化和棋谱导出功能的五子棋游戏。为了践行面向对象的设计思想，我们将程序拆分成了多个各司其职的类，保证了代码的可读性和低耦合，全项目总代码量达到了 **1200 行以上**。

## 🌟 核心功能
* **多种博弈模式**：支持“双人对战”、“简单人机（随机落子）”、“中等人机（偏向进攻）”和“困难人机（攻守兼备打分算法）”。
* **战绩与棋谱存储**：利用 Java `Properties` 实现了战绩在本地硬盘的永久保存；并支持通过原生 IO 流将对局画面以字符画形式导出为 `board_log.txt` 文本。
* **高交互体验**：提供菜单栏、专用的独立规则说明弹窗（HelpDialog），以及防误触的智能悔棋功能。
* **精美UI渲染**：棋盘外侧附带标准的 A-O, 1-15 坐标刻度系；棋子使用 `RadialGradientPaint` 径向渐变算法，渲染出立体的 3D 玻璃质感。

## 🧑‍🤝‍🧑 团队分工（每人代码量均达标 400+ 行）
本项目由 3 人团队合作完成，具体职责划分如下：

* **[成员 A] 算法与逻辑模块**：负责 `GameModel.java` 与 `AIPlayer.java`。
  * **贡献**：搭建了底层的数据模型，实现了安全防越界的落子与历史记录悔棋；编写了分向独立探测的五连珠胜负判定算法；实现了通过循环打分评估防守和进攻价值的电脑 AI。
  
* **[成员 B] UI与主控模块**：负责 `GomokuGame.java` 与 `HelpDialog.java`。
  * **贡献**：负责整个应用的主窗口流程调度，利用边界布局与盒式布局组装了右侧控制面板；通过引入 `Timer` 异步延迟解决了人机对弈时界面瞬间卡顿的问题；独立编写了模态游戏规则帮助弹窗。
  
* **[成员 C / 组长] 视图组件与存储模块**：负责 `BoardPanel.java`、`RecordManager.java` 和 `BoardExporter.java`。
  * **贡献**：负责重写底层画板，实现鼠标屏幕像素坐标到数组下标的四舍五入映射；完成了立体棋子和坐标刻度的图形绘制；负责本地 `.properties` 配置文件的安全读写，以及双重循环输出 `.txt` 棋谱。

## 🚀 如何运行
1. 确保您的电脑已配置好 Java 运行环境 (JDK 8 或以上)。
2. 获取本仓库代码，在根目录下可以找到已经打包好的 `GomokuGame.jar` 文件（或者直接通过 IDE 运行 `GomokuGame.java`）。
3. **直接双击** 该 `.jar` 文件即可畅玩；
   *(或者在 CMD/终端中输入：`java -jar GomokuGame.jar`)*
